# Day 4: MBTI 성격 유형 테스트 웹 앱 제작 일지 및 사양서

본 문서는 Day 4에 진행된 **MBTI 성격 유형 테스트 웹 앱** 프로젝트의 개발 내용, 아키텍처 및 상세 사양을 후속 유지보수와 복습을 위해 상세히 정리한 사양서입니다.

---

## 1. 프로젝트 개요
- **목적**: 사용자가 8개의 직관적인 무작위 질문에 답하여 자신의 진정한 MBTI 성격 유형을 분석해 주는 프리미엄 디자인의 정적 웹 애플리케이션입니다.
- **주요 기능**:
  1. 글래스모피즘(Glassmorphism) 기반의 트렌디하고 세련된 다크 테마 UI/UX.
  2. 총 16개 질문 풀(각 지표당 4문항)에서 균형 있게 8문항(각 지표당 2문항)을 랜덤 추출 및 셔플하는 시스템.
  3. 선택지 순서 및 최종 문항 리스트의 셔플 처리.
  4. AI 이미지 생성 및 Python 자동 크로핑(Cropping)을 연계한 16종 전용 MBTI 캐릭터 일러스트 탑재.
  5. 부드러운 스크린 전환 애니메이션 및 결과 연산 전 "성격 분석 중..." 로딩 경험 선사.

## 2. 파일 및 폴더 구조
```text
Day_4/
├── index.html        # 웹 앱 골격 및 뼈대 마크업 (시작 화면, 질문 화면, 결과 화면 통합)
├── style.css         # 글로우 백그라운드, 애니메이션, 카드 레이아웃, 글래스모피즘 스타일링
├── app.js            # 질문 풀 데이터, 랜덤 연산, MBTI 계산, 화면 전환 및 결과 노출 로직
├── slice_grid.py     # 원본 4x4 그리드 캐릭터 이미지 분할 파이썬 스크립트

    ├── enfj.png, enfp.png, entj.png, entp.png ...
    └── (총 16개의 캐릭터 이미지)
```

## 3. 핵심 기능별 상세 메커니즘

### 3.1. 문제 밸런싱 및 셔플링 알고리즘 (`app.js`)
문제가 매번 다르게 나오도록 무작위성을 주면서도, 특정 MBTI 지표에 치우쳐 신뢰도를 떨어뜨리지 않도록 **정교한 지표별 균등 추출 알고리즘**을 사용했습니다.
1. **그룹별 추출**: E/I, S/N, T/F, J/P 각각 4문항씩 준비된 풀(Pool)에서 개별적으로 무작위 2문항씩을 먼저 슬라이싱해 총 8문항을 수집합니다.
2. **문항 셔플**: 지표 순서(예: E/I 순서로 쭉 진행되는 것)를 깨기 위해 선별된 8문항을 피셔-예이츠 셔플 알고리즘으로 다시 한 번 무작위 정렬합니다.
3. **선택지 셔플**: 버튼 위치가 고착화되는 것을 방지하기 위해, 각 문제의 두 선택지 순서도 렌더링 시 매번 랜덤하게 뒤집습니다. 이 과정에서 각 버튼과 연결된 가중치(예: I 또는 E점수 가산)도 연동하여 셔플되므로 분석의 정밀도는 그대로 보존됩니다.

### 3.2. 일러스트 크로핑 자동화 (`slice_grid.py`)
- AI 이미지 생성 제약을 지혜롭게 극복하고자, **16개의 마스코트 캐릭터가 담긴 단일 4x4 그리드 이미지**를 생성했습니다.
- 작성된 파이썬 스크립트는 `Pillow` 라이브러리를 활용하여 이미지 해상도(1024x1024)를 균등한 4행 4열로 분할(각 256x256 픽셀)하고, 아래 순서대로 16개의 매칭 이미지 파일로 변환하여 `images/` 폴더에 배포합니다:
  - 1행: ISTJ, ISFJ, INFJ, INTJ
  - 2행: ISTP, ISFP, INFP, INTP
  - 3행: ESTP, ESFP, ENFP, ENTP
  - 4행: ESTJ, ESFJ, ENFJ, ENTJ

### 3.3. 디자인 시스템 및 인터랙션 요소 (`style.css`)
- **다크 네온 테마**: 백그라운드에 세 개의 흐릿하고 커다란 보라색/분홍색 원형 그라데이션 레이어(`.bg-glow`)가 천천히 부유(`float-glow`)하며 우주와 같은 깊고 세련된 느낌을 제공합니다.
- **유리 재질(Glassmorphism)**: 결과 카드와 버튼에 `backdrop-filter: blur(20px)`와 미세한 투명 테두리(`border: 1px solid rgba(255, 255, 255, 0.08)`)를 주어 반사 유리의 질감을 시각화했습니다.
- **마이크로 애니메이션**:
  * 선택지 호버 시 카드가 부드럽게 위로 들리며(`translateY(-2px)`) 네온 광채를 강하게 내뿜는 효과.
  * 결과 화면의 메인 캐릭터 이미지가 아래위로 천천히 흔들리는 부유 효과(`float-char`).
  * "분석 중..." 화면에서 궤도를 따라 도는 로딩 서클 및 은은하게 펄스하는 후광 발광 연출.

## 4. 수록된 16가지 성향 질문 목록

| 분류 | 번호 | 질문 질문 | 선택지 1 (가산점) | 선택지 2 (가산점) |
| :--- | :--- | :--- | :--- | :--- |
| **E / I** | Q1 | 주말에 약속이 없다면? | 집에 있는다 (I) | 밖에 나가서 사람들을 만난다 (E) |
| | Q2 | 새로운 사람들과 처음 만났을 때 나는? | 조용히 경청하며 말을 아낀다 (I) | 먼저 말을 걸며 대화를 주도한다 (E) |
| | Q3 | 스트레스를 많이 받았을 때 풀 수 있는 방법은? | 혼자만의 시간을 가지며 에너지를 충전한다 (I) | 친구들과 수다를 떨거나 모임에 참석한다 (E) |
| | Q4 | 오랜만에 온 휴가날, 나는 어디로 향할까? | 조용하고 아늑한 독채 펜션이나 집 (I) | 축제나 핫플레이스 등 활기찬 장소 (E) |
| **S / N** | Q5 | 영화를 보거나 소설을 읽을 때 나의 생각은? | 스토리 라인과 현실적 묘사에 집중한다 (S) | 주인공이나 결말 이후 등 엉뚱한 상상을 펼친다 (N) |
| | Q6 | “하늘을 나는 생각을 가끔 해” 라는 말을 들으면? | “그게 무슨 소리야? 불가능하잖아” 라고 생각한다 (S) | “오, 날아다니면 무슨 기분일까?” 같이 공상에 빠진다 (N) |
| | Q7 | 새로운 요리 레시피를 보고 만들 때 나는? | 레시피의 계량과 순서를 정확하게 지키며 요리한다 (S) | 대충 눈대중으로 보고 내 느낌대로 변형해서 요리한다 (N) |
| | Q8 | 만약 아무 일도 일어나지 않는 조용한 숲속을 걷는다면? | 풀 냄새, 바람 소리, 나무의 질감 등 감각을 고스란히 느낀다 (S) | 숲속에 요정이 살지 않을까? 혹은 미래에 대해 생각한다 (N) |
| **T / F** | Q9 | 일이 뜻대로 되지 않아 고민하는 친구에게 나는? | 무엇이 문제인지 현실적인 해결책을 제시해 준다 (T) | 친구의 속상한 마음에 깊이 공감하고 위로해 준다 (F) |
| | Q10| 친구와 다툼이 생겼을 때 나는? | 왜 갈등이 생겼는지 논리적으로 따지고 분석한다 (T) | 갈등으로 상처받은 마음과 감정을 먼저 풀고 싶다 (F) |
| | Q11| 친구가 “나 힘들어서 화분 샀어” 라고 말했을 때 나의 반응은? | “무슨 화분 샀어? 기르기 쉬운 거야?” 물어본다 (T) | “많이 힘들었구나... 무슨 일 있었어?” 위로한다 (F) |
| | Q12| 어떤 중요한 선택을 해야 할 때, 나에게 더 와닿는 기준은? | 인과관계가 확실하고 객관적으로 타당한 사실 (T) | 관계되는 사람들의 감정과 내면의 가치관 (F) |
| **J / P** | Q13| 여행 계획을 세울 때 나의 스타일은? | 시간 단위로 일정을 꼼꼼하게 다 계획해 둔다 (J) | 대략적인 목적지만 정하고 상황에 따라 자유롭게 다닌다 (P) |
| | Q14| 주말 과제를 할 때 나의 행동은? | 미리미리 계획적으로 나누어 끝내 둔다 (J) | 미루다가 마감 직전에 한꺼번에 집중해서 한다 (P) |
| | Q15| 약속 당일 갑자기 계획이 취소된다면? | 할 일이 붕 떠버려서 아쉽고 조금 스트레스를 받는다 (J) | “오히려 좋아!” 하며 다른 하고 싶었던 일을 즉흥적으로 한다 (P) |
| | Q16| 방 정리를 하거나 물건을 정돈할 때 나는? | 제자리가 딱 정해져 있고 체계적으로 정돈되어야 편안하다 (J) | 대충 편리한 곳에 두고 필요할 때 찾아서 쓴다 (P) |

## 5. 실행 및 로컬 서빙 방법
1. 로컬 환경에 파이썬이 설치되어 있는지 확인합니다.
2. 터미널을 열고 `Day_4` 폴더 위치로 이동합니다.
3. 아래 명령어를 실행하여 웹 서버를 엽니다:
   ```bash
   python -m http.server 58151
   ```
4. 크롬 등 브라우저를 열고 `http://localhost:58151`에 접속하여 원활하게 작동하는 화면을 확인합니다.
